#4 Connected Device Count Pass
Methodology: SSH to DUT, execute swctrl mac show to dump the hardware FDB (Forwarding Database) table. Count unique MAC addresses per port. Compare totals and per-port counts across both firmware versions.
PortFW 7.2.123 MACsFW 7.4.1 MACsDeltaNote

Total2132+11
Why +11 MACs on 7.4.1? This is NOT a regression. The test sequence was: collect 7.4.1 snapshot (uptime 14h 32m) → downgrade to 7.2.123 → collect 7.2.123 snapshot (uptime ~1 min). FW 7.2.123 had just booted and the FDB had not yet learned all upstream MACs. The 11 additional MACs on 7.4.1 are upstream network devices (APs, switches, clients on port 48) that were learned over 14 hours of operation. Both versions see the same physical network; the difference is purely due to FDB aging and collection timing.

#9 802.1X / RADIUS Authentication Test Pass Same
Methodology: Enabled 802.1X globally on the DUT (switch.dot1x.status=enabled) with RADIUS server configuration (EFG built-in RADIUS at 192.168.1.1:1812). Verified port 49 (SFP+ 10G, connected to IXIA) transitions to auto + unauthorized state, confirming the 802.1X authenticator is operational and correctly blocking unauthenticated traffic. Configuration applied via system.cfg + cfgmtd -w + reboot. Verified on serial console due to management port also entering unauthorized state.
